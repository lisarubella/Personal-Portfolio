## Deployment Steps

### Step 1: Final Local Tests

```bash
# Clean install
rm -rf node_modules package-lock.json
npm install

# Type check
npm run type-check

# Lint
npm run lint

# Build production
npm run build

# Test production build
npm start
# Visit http://localhost:3000
# Verify everything works
# Ctrl+C to stop
```

### Step 2: Commit to Git

```bash
git init
git add .
git commit -m "feat: Initial portfolio production build

- Next.js 14 with TypeScript
- Tailwind CSS with glassmorphism design
- View Transition API theme toggle
- Responsive mobile-first design
- 4 featured projects
- Dark mode support
- SEO optimized
- Production ready for Vercel deployment"

git remote add origin https://github.com/yourusername/mona-portfolio
git branch -M main
git push -u origin main
```

### Step 3: Deploy to Vercel

#### Method A: CLI Deployment (Fastest)

```bash
# Install Vercel CLI (one-time)
npm i -g vercel

# Deploy from project directory
vercel

# Follow prompts:
# - Link to Vercel account
# - Choose "Link to existing project?" → No
# - Choose "What's your project's name?" → mona-portfolio
# - Choose framework → Next.js
# - Build command → npm run build
# - Output directory → .next
# - Environment variables → Skip (none needed)

# After deployment:
# Your site will be live at https://mona-portfolio.vercel.app
```

#### Method B: GitHub + Vercel Dashboard

1. Push to GitHub (see Step 2)
2. Go to https://vercel.com/dashboard
3. Click "Add New..." → "Project"
4. Select GitHub repository "mona-portfolio"
5. Click "Import"
6. Vercel auto-detects Next.js:
   - Framework: Next.js ✓
   - Build Command: npm run build ✓
   - Output Directory: .next ✓
7. Click "Deploy"
8. Wait for build (usually 2-3 minutes)
9. Get deployment URL and custom domain setup

### Step 4: Post-Deployment Verification

```
After deployment URL is provided:

[ ] Visit deployment URL
[ ] Homepage loads without errors
[ ] Theme toggle works
[ ] Mobile menu works on small screens
[ ] All navigation links work
[ ] Project links open correctly
[ ] Email link works (mailto:)
[ ] Performance is good (< 3s load time)
[ ] No console errors (DevTools F12)
[ ] Mobile responsive (use Chrome DevTools)
[ ] Dark mode works
```

### Step 5: Custom Domain (Optional)

If you have a custom domain (e.g., mona.com):

1. In Vercel Dashboard → Project Settings
2. Go to "Domains" section
3. Add your domain
4. Update DNS records (follow Vercel's instructions)
5. Wait for DNS propagation (usually instant)

### Step 6: Set Up Analytics (Optional)

**Vercel Analytics** (Recommended - Free):
1. Dashboard → Project Settings → Analytics
2. Enable "Web Analytics"
3. Analytics appear after 5-10 minutes

**Google Analytics** (Optional):
1. Create GA4 property: https://analytics.google.com
2. Get measurement ID (G-XXXXXXXXXX)
3. Add to `.env.local`:
   ```env
   NEXT_PUBLIC_GA_ID=G-XXXXXXXXXX
   ```
4. Create `components/analytics.tsx` with gtag script
5. Import in `app/layout.tsx`

## Rollback Procedure

If something goes wrong:

**In Vercel Dashboard**:
1. Go to "Deployments"
2. Find last good deployment
3. Click "..." menu
4. Select "Promote to Production"
5. Verify at https://your-domain.com
